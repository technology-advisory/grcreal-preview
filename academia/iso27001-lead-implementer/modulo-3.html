Modulo 3

GAP Analysis: donde estamos y donde debemos estar

Antes de planificar el viaje, hay que saber el punto de partida.
El GAP Analysis es el diagnostico que lo hace posible.

Definicion
Que es un GAP Analysis
Un GAP Analysis consiste en un analisis de cumplimiento tanto de los requisitos de ISO/IEC 27001 como de sus controles del Anexo A. Es la herramienta que permite identificar la diferencia entre el estado actual de la organizacion y los requisitos que debe cumplir para obtener la certificacion.
“El GAP Analysis responde a tres preguntas: donde estamos, donde debemos estar, y que nos falta para llegar.”
Metodologia CertiProf
Sin un GAP Analysis riguroso, cualquier plan de implementacion se construye sobre suposiciones. El resultado: plazos irreales, recursos insuficientes y sorpresas desagradables durante la auditoria.
Objetivos
Para que sirve el GAP Analysis
1 Identificar las brechas entre el estado actual y los requisitos de ISO 27001
2 Priorizar las acciones necesarias para alcanzar la conformidad
3 Estimar el esfuerzo, tiempo y recursos necesarios para la implementacion
4 Establecer una linea base para medir el progreso durante el proyecto
5 Proporcionar a la direccion informacion objetiva para la toma de decisiones
Metrica
Modelos de madurez: como medimos
Para realizar el GAP Analysis, es recomendable utilizar un modelo de madurez que permita evaluar el cumplimiento de forma objetiva. Los modelos mas comunes incluyen:
COBIT Modelo de madurez de ISACA, ampliamente utilizado para gobierno y gestion de TI
NIST Framework de ciberseguridad con niveles de implementacion progresivos
ISM3 Modelo de madurez de seguridad de la informacion especifico
SSE-CMM Capability Maturity Model para ingenieria de seguridad de sistemas
Escala
Niveles de madurez COBIT
0 Inexistente. La gestion de procesos no se aplica en absoluto
1 Inicial / Ad-hoc. Los procesos son desorganizados y reactivos
2 Repetible. Los procesos siguen un patron regular pero no estandarizado
3 Definido. Los procesos estan documentados y comunicados
4 Gestionado / Medible. Los procesos son monitorizados y medidos
5 Optimizado. Las buenas practicas son utilizadas y automatizadas, con mejora continua
Este modelo se adapta perfectamente para establecer un nivel de madurez actual y un nivel objetivo para cada uno de los requisitos (clausulas) y controles (Anexo A) de ISO/IEC 27001.
Metodologia
Como realizar un GAP Analysis
El proceso tipico para realizar un GAP Analysis incluye los siguientes pasos:
1 Definir el alcance. Determinar que areas, procesos y ubicaciones se incluyen en el analisis
2 Seleccionar el modelo de madurez. Elegir la escala y criterios de evaluacion
3 Recolectar evidencias. Revisar documentacion existente, entrevistar personal, observar procesos
4 Evaluar cada requisito. Asignar un nivel de madurez actual a cada clausula y control relevante
5 Identificar brechas. Comparar el nivel actual con el nivel requerido para la certificacion
6 Documentar hallazgos y recomendar acciones. Priorizar las brechas segun su criticidad
Error tipico
El GAP Analysis que no es un analisis
“Ya sabemos lo que nos falta. No hace falta un analisis formal.”
Frase que precede a un proyecto con sobrecostes y plazos incumplidos.
El error mas frecuente es saltarse el GAP Analysis o hacerlo superficialmente, asumiendo que "se conoce" el estado de la organizacion. Un GAP Analysis basado en suposiciones, sin evidencias objetivas, no es un analisis — es una opinion documentada. Y las opiniones no sirven para planificar un proyecto de implementacion.
❌ Enfoque incorrecto: "Cumplimos la mayoria de los requisitos. Solo falta documentar algunas politicas."
✅ Enfoque correcto: "Tras evaluar 87 controles con evidencias objetivas, tenemos un cumplimiento del 42% en categoria Media y del 18% en categoria Alta."
Resumen del modulo
Lo que debes llevarte de aqui
1 El GAP Analysis es un analisis de cumplimiento de requisitos y controles de ISO 27001
2 Utiliza modelos de madurez (como COBIT) para evaluar objetivamente el estado actual
3 Identifica brechas, las prioriza y permite estimar esfuerzo y recursos
4 Sin un GAP Analysis riguroso, el plan de implementacion se construye sobre suposiciones
